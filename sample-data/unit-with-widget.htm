Widget Konzept

Mit dieser Unit soll das Konzept für Verona-Widgets erklärt werden. Sie verwendet den simple player 6.0 in einer modifizierten Version. Es handelt sich lediglich um eine Konzeptskizze, daher werden auch mögliche Alternativen genannt.

Was ist ein Verona Widget?

Es handelt sich um ein, von der eigentlichen Unit unabhängiges interaktives Element, das

  • Unit-übergreifend verwendet werden kann und seinen Zustand unit-übergreifend behalten kann.
  • möglichst genau wie die player im Booklet-Ladevorgang geladen werden kann.
  • als Overlay über der eigentlichen Aufgabe (und sogar dem Host?) erscheinen kann.

Grundsätzlich verwenden Widgets dieselbe Technologie wie Verona-PLayer an sich: ein iframe, das srcdoc Attribut und postMessages.

Darstellung

Vorschlag: Widgets werden grundsätzlich als Vollbildoverlay dargestellt. Sie können aber den Host darüber informieren, sich beschneiden zu lassen (Details s. U.), sodass die darunter liegenden Elemente erreichbar sind. Dafür kann sie css-eigenschaft clip-path verwendet werden. Widgets müssen daher nicht rechteckig sein. :

Sollte das Widget verschiebbar sein sollen, ist die Drag-Und-Drop-Funktionalität im Widget selbst umzusetzen und im Verschiebungsfall der Host über einen veränderten clip-path zu informieren. Beispiel:

Analog dazu entscheidet auch der Widget-Autor selbst darüber, ob es z. B. einen schließen Button gibt.

Das Konzept "immer fullscreen overlay und clip path" ist noch nicht im Zusammenspiel mit Mobile Devices erprobt.

Alternativ könnten a priori verschiedene Darstellungsweisen - z. B. "modal", "overlay", "sidebar", "movable" etc. festgelegt werden. Beim Aufruf des Widgets wird eine davon gesetzt. Für den Widgetautor würde das weniger Implementationsaufwand bedeuten, aber auch sehr viel weniger Funktionalität. Drag & Drop würde dann ggf. im Host umgesetzt werden. Das Widget müsste dennoch seine Größe selbst an den Host melden. Es könnte dann nur rechteckig sein. Eine Kombination aus beiden Konzepten ist evenfalls denkbar.
